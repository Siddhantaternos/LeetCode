# 🔗 Combination Sum 

## 📌 Problem Summary

Given:

* An array `candidates` of **distinct positive integers**
* A target integer `target`

Return **all unique combinations** of candidates where the chosen numbers sum to `target`.

* You may **reuse** candidates unlimited times.
* Combinations must be returned in **any order**.
* Two combinations are considered the same if the elements are the same regardless of order.

---

## 🧠 Core Insight

This is a classic **Backtracking** problem.

The key is:

* Explore all possible combinations
* Only accept combinations whose sum equals `target`
* Avoid duplicates by **fixing order** during recursion

This is not typical DP, not greedy — it’s **DFS tree exploration with pruning**.

---

## 🎯 Strategy

1. **Sort** the candidate array (optional but helps pruning).
2. Use recursive **backtracking**:

   * Keep a **path list** representing current combination
   * Keep a **remaining sum** (`remaining`)
   * At each step, you can:

     * Include the current candidate again (since repetition is allowed)
     * Move to the next candidate
3. Stop recursion when:

   * `remaining == 0` → found a valid combination → add to answer
   * `remaining < 0` → overshot → backtrack

---

## ⛏ Important Pattern Notes

* This is a **combination generation** problem → no order matters inside the combination itself
* To avoid duplicates:

  * You never revisit indices lower than the current index in recursion
  * This guarantees combinations are built in **non-decreasing order**

---

## 🧠 Python Code (LeetCode Style)

```python
from typing import List

class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        result = []
        candidates.sort()  # optional but helpful

        def backtrack(start, path, remaining):
            if remaining == 0:
                result.append(path.copy())
                return
            if remaining < 0:
                return

            for i in range(start, len(candidates)):
                num = candidates[i]
                # Choose
                path.append(num)
                # Continue with i (not i+1) because we can reuse
                backtrack(i, path, remaining - num)
                # Unchoose
                path.pop()

        backtrack(0, [], target)
        return result
```

---

## ⏱️ Complexity

| Metric | Value                                           |
| ------ | ----------------------------------------------- |
| Time   | Exponential (backtracking)                      |
| Space  | O(target / min(candidates)) for recursion depth |

Even though brute force is exponential, recursion prunes early and avoids generating invalid paths unnecessarily.

---

## 🧠 Mental Model

```
Fix a candidate →Branch deeper→
If sum == target → accept
If sum > target → stop
Else → try more from current index
```

This is a tree search — every path is a potential valid combination.
