# 🧩 Sudoku Solver

## 📌 Problem Overview

You’re given a partially filled 9×9 Sudoku board. Empty cells are marked with `"."`. The task is to fill them so that:

* Every **row** contains digits `1–9` exactly once
* Every **column** contains digits `1–9` exactly once
* Every **3×3 sub-box** contains digits `1–9` exactly once

The board is always 9×9. Standard Sudoku rules apply.

---

## 🧠 Key Idea

This is a classic **constraint satisfaction** problem.

Each empty cell must satisfy three constraints simultaneously:

* Row validity
* Column validity
* 3×3 box validity

There’s no greedy shortcut here. The correct approach is **Backtracking (DFS)** with careful validation at each step.

---

## 💡 Strategy (Backtracking)

1. Locate an empty cell (`"."`)
2. Try placing digits `'1'` through `'9'`
3. If the placement is valid:

   * Fill it
   * Recurse to solve the rest
4. If none work:

   * Undo the placement (backtrack)
5. When no empty cells remain, the board is solved

This is depth-first search with pruning.

---

## ✔️ Validity Rules

When placing digit `d` at position `(r, c)`:

* It must not already appear in:

  * The same row
  * The same column
  * The same 3×3 box

To identify the box:

```
boxIndex = (r // 3) * 3 + (c // 3)
```

This uniquely maps each cell to one of the 9 sub-boxes.

---

## ⚡ Optimization: O(1) Validation

Instead of scanning rows, columns, and boxes repeatedly, we maintain sets:

```
rows[r]
cols[c]
boxes[boxIndex]
```

Each stores digits already used.

This makes every validity check constant time.

## 🧠 Python Implementation

```python
from typing import List

class Solution:
    def solveSudoku(self, board):
        rows = [set() for _ in range(9)]
        cols = [set() for _ in range(9)]
        boxes = [set() for _ in range(9)]
        empties = []

        # Initialize structures
        for r in range(9):
            for c in range(9):
                if board[r][c] == ".":
                    empties.append((r, c))
                else:
                    val = board[r][c]
                    rows[r].add(val)
                    cols[c].add(val)
                    boxes[(r // 3) * 3 + (c // 3)].add(val)

        def backtrack():
            if not empties:
                return True

            # Choose cell with fewest possible options (MRV heuristic)
            min_options = 10
            min_index = -1

            for i, (r, c) in enumerate(empties):
                box = (r // 3) * 3 + (c // 3)
                options = 9 - len(rows[r] | cols[c] | boxes[box])
                if options < min_options:
                    min_options = options
                    min_index = i
                if min_options == 1:
                    break

            r, c = empties.pop(min_index)
            box = (r // 3) * 3 + (c // 3)

            for d in map(str, range(1, 10)):
                if d not in rows[r] and d not in cols[c] and d not in boxes[box]:
                    board[r][c] = d
                    rows[r].add(d)
                    cols[c].add(d)
                    boxes[box].add(d)

                    if backtrack():
                        return True

                    board[r][c] = "."
                    rows[r].remove(d)
                    cols[c].remove(d)
                    boxes[box].remove(d)

            empties.insert(min_index, (r, c))
            return False

        backtrack()
```

---

## ⏱ Complexity

* **Time:** Exponential in the worst case
* **Space:** O(1) extra space (board modified in-place)

Backtracking is inherently exponential, but pruning with row/column/box sets — and choosing the most constrained cell first — dramatically reduces the search space.

## 🧠 Conceptual Model

Sudoku solving boils down to:

```
DFS + Constraint Validation
```

The flow is simple:

* Explore
* Validate
* Commit
* Undo if invalid

This pattern appears in N-Queens, crossword solvers, logic puzzles, and many constraint-based systems.
